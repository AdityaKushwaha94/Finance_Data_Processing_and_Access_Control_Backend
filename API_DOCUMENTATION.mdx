# Finance Dashboard API Documentation

This file provides complete endpoint-level documentation with dummy request and response examples.

## Base URL

`http://localhost:4000`

## Authentication Model

- Auth type: Bearer JWT
- Header format:

```http
Authorization: Bearer <your_jwt_token>
```

## Role Matrix

- `viewer`: can access dashboard overview only
- `analyst`: can read records and dashboard overview
- `admin`: full access to users, records, and dashboard

## 2) Auth APIs

### POST /api/auth/register

- Access: Public
- Purpose: Register a new user (default role = `viewer`)

#### Request Body

```json
{
  "name": "Aarav Verma",
  "email": "aarav@example.com",
  "password": "User@12345"
}
```

#### Example Response (201)

```json
{
  "success": true,
  "message": "User registered successfully",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.sample",
    "user": {
      "id": "665f3b1f2a8e4c7d99f00111",
      "name": "Aarav Verma",
      "email": "aarav@example.com",
      "role": "viewer",
      "isActive": true,
      "createdAt": "2026-04-05T10:00:00.000Z",
      "updatedAt": "2026-04-05T10:00:00.000Z"
    }
  }
}
```

#### Validation Error Example (400)

```json
{
  "success": false,
  "message": "Invalid request body",
  "details": {
    "formErrors": [],
    "fieldErrors": {
      "password": [
        "Password must be at least 8 characters"
      ]
    }
  }
}
```

## 3) User Management APIs (Admin only)

> All endpoints below require an `admin` token.

- Access: `admin`

```json
{
  "success": false,
  "message": "Invalid or expired authorization token"
}
```

### Forbidden (403)

```json
{
  "success": false,
  "message": "Forbidden: you do not have access to this resource"
}
```

### Route Not Found (404)

```json
{
  "success": false,
  "message": "Route not found: GET /api/unknown"
}
```

### Invalid ObjectId (400)

```json
{
  "success": false,
  "message": "Invalid request params",
  "details": {
    "formErrors": [],
    "fieldErrors": {
      "id": [
        "Invalid MongoDB ObjectId"
      ]
    }
```

## 7) Quick Postman Sequence (Suggested)

1. `POST /api/auth/login` with default admin credentials from `.env`
2. Use token in Authorization header
3. `POST /api/users` to create analyst/viewer users
4. `POST /api/records` to create income/expense entries
5. `GET /api/records` with filters/pagination
6. `GET /api/dashboard/overview` for analytics
